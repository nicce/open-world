# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7: Combat Wiring + HUD Strip.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This is a Godot Engine 4.2+ game project. There is no web framework, no shadcn, and no
> CSS. All design tokens map to Godot's built-in Control node properties, Theme overrides,
> and inline GDScript values. The shadcn gate and registry gate are not applicable.

| Property | Value |

## Spacing Scale

Godot uses pixels directly; tokens below map to pixel values used in scene properties (`custom_minimum_size`, `position`, `offset_*`, `separation`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon inner padding inside HUD slot, indicator label padding |
| sm | 8px | Gap between weapon slot and tool slot in HUD strip HBoxContainer (`separation`) |
| md | 16px | HUD strip bottom margin from viewport edge (anchor offset) |
| lg | 24px | WeaponIndicator position offset above player origin (`Vector2(0, -24)`) |
| xl | 32px | HUD strip horizontal outer padding from viewport center |
| 2xl | 48px | HUD slot minimum size (width and height: 48x48px) |

Exceptions:
- Inventory slots are 87x80px (established in Phase 2, not changed by this phase).
- HUD slots are intentionally smaller than inventory slots: 48x48px. HUD is a compact
  persistent strip, not the full inventory panel.

**Source:** Existing inventory slot size from `scenes/inventory_slot_ui.tscn`
(`custom_minimum_size = Vector2(87, 80)`). HUD slots set at 48x48px — default decision,
matches compact RPG hotbar convention cited in CONTEXT.md.

## Typography

Godot default theme font at default size unless overridden. Only Label nodes need
explicit size overrides where readability requires it.

| Role | Size | Weight | Line Height | Godot Property |
|------|------|--------|-------------|----------------|
| HUD slot label ("W" / "T") | 11px | regular (400) | 1.2 | `add_theme_font_size_override("font_size", 11)` |
| HUD slot fallback item name | 9px | regular (400) | 1.2 | `add_theme_font_size_override("font_size", 9)` |
| WeaponIndicator item name (fallback) | 9px | regular (400) | 1.2 | `add_theme_font_size_override("font_size", 9)` |
| Existing inventory UI labels | Godot default | regular (400) | default | unchanged — do not modify |

**Constraints:**
- Maximum 2 font sizes introduced by this phase (11px slot label, 9px item fallback).
- Inventory UI labels (`WeightLabel`, `AbbrevLabel`, `QuantityLabel`) are not modified.
- WeaponIndicator text must be legible at the small scale of the player sprite; 9px is
  the minimum — do not go smaller.

**Source:** RESEARCH.md recommends 9px matching existing `collectable.gd` label scale
(noted in `fix(collectable): reduce label font size to 6px` commit — actual in-game
scale is ~62% smaller due to node scaling; the Label node itself should be set to 9px
in scene properties, not script).

## Color

All colors expressed as Godot `Color(r, g, b, a)` values or hex strings. No CSS.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `Color(0, 0, 0, 0)` transparent / game world | Game world surface behind all HUD elements |
| Secondary (30%) | `Color(0.12, 0.12, 0.12, 0.85)` dark charcoal, 85% opaque | HUD slot panel background (filled and empty), HUD strip backing |
| Accent (10%) | `Color(1.0, 0.85, 0.0, 1.0)` gold — `#FFD900` | HUD slot border when occupied (2px StyleBoxFlat border) |
| Dimmed empty state | `Color(1, 1, 1, 1)` with `modulate.a = 0.4` | Empty HUD slot — same pattern as inventory slot empty state |
| WeaponIndicator background | none (no background panel) | Indicator is label/texture only, no backing box |

Accent reserved for:
- HUD slot border when the slot is occupied (weapon or tool equipped).
- Slot border is 2px, matches the existing inventory selection border width.
- Accent is NOT used for the slot border when the slot is empty.
- Accent is NOT used for the WeaponIndicator node.

Dimming implementation: `modulate.a = 0.4` on the slot Panel node when empty,
`modulate.a = 1.0` when occupied. This is the confirmed pattern from
`scripts/inventory_slot_ui.gd` lines 50 and 56.

**Source:** CONTEXT.md — "Empty slots: darker/dimmed appearance (visually distinct from
filled slots)." RESEARCH.md — "Slot dimming: `modulate = Color(0.4, 0.4, 0.4)` — one-liner."
Existing accent color from `inventory_slot_ui.gd` line 25: `Color(1.0, 0.85, 0.0)`.

## Component Inventory

Components new to Phase 7. Each maps directly to a Godot scene or script node.

### HUD Strip (HUD-01, HUD-02)

**Scene:** `scenes/hud_strip.tscn`
**Script:** `scripts/hud_strip.gd`
**Base node:** `Control`
**Placement:** Direct child of world's `CanvasLayer`, sibling of `InventoryUI` — NOT a child of `InventoryUI`.

```
Control (HudStrip)                         # hud_strip.tscn root
  HBoxContainer                            # weapon slot + tool slot side by side
    VBoxContainer (WeaponSlotGroup)
      Panel (WeaponSlot)                   # 48x48px slot box; Panel for StyleBox border
        TextureRect (IconRect)             # item texture; fill slot; visible when item has texture
        Label (NameLabel)                  # item name abbreviation; visible when no texture
    VBoxContainer (ToolSlotGroup)
      Panel (ToolSlot)                     # 48x48px slot box
        TextureRect (IconRect)
        Label (NameLabel)
  (slot labels "W" and "T" are placed below their VBoxContainer via second Label child)
```

**Anchor:** Bottom center of viewport.
- `anchor_left = 0.5`, `anchor_right = 0.5`, `anchor_top = 1.0`, `anchor_bottom = 1.0`
- `offset_left = -52px`, `offset_right = 52px` (accounts for 2x 48px slots + 8px gap)
- `offset_bottom = -16px` (16px bottom margin from viewport edge)

**Slot panel styling:**
- `custom_minimum_size = Vector2(48, 48)`
- `StyleBoxFlat` background: `bg_color = Color(0.12, 0.12, 0.12, 0.85)`
- Empty slot: `modulate.a = 0.4`; no border
- Occupied slot: `modulate.a = 1.0`; 2px border `Color(1.0, 0.85, 0.0)`

**Slot label ("W" / "T"):**
- `Label` node inside `VBoxContainer`, below the `Panel`
- Font size: 11px
- `horizontal_alignment = HORIZONTAL_ALIGNMENT_CENTER`
- Text: `"W"` for weapon slot, `"T"` for tool slot

**Icon display (texture-or-label pattern — verbatim from collectable.gd):**
```gdscript
if item and item.texture:
    icon_rect.texture = item.texture
    icon_rect.visible = true
    name_label.visible = false
elif item:
    name_label.text = item.name
    name_label.visible = true
    icon_rect.visible = false
else:
    icon_rect.visible = false
    name_label.visible = false
```

**Signal subscription:** Connect to `equipment_data.equipment_changed` in `set_equipment_data()`.
Call `_on_equipment_changed()` immediately after connecting to paint initial state.

### WeaponIndicator (CMBT-05)

**Node placement:** Child of Player root node in `scenes/player.tscn`.
**Base node type:** `Node2D` container.

```
Node2D (WeaponIndicator)                   # position = Vector2(0, -24); visibility toggled
  TextureRect (IconRect)                   # item texture; expand_mode = 1; stretch_mode = 6
  Label (NameLabel)                        # item name fallback; font_size = 9px
```

**Position:** `Vector2(0, -24)` above player origin. Adjust in Inspector after first run if indicator overlaps player sprite.

**Visibility rule:** `visible = weapon != null` — the entire `Node2D` is hidden when no weapon is equipped. No partial hidden state.

**Sizing:** `TextureRect` minimum size `Vector2(16, 16)`. No background panel.

**Font size:** 9px on `NameLabel`.

**Signal subscription:** `equipment_data.equipment_changed.connect(_on_equipment_changed)`.
Call `_on_equipment_changed()` at end of `_ready()` to paint initial state (handles case where
`equipment_data` already has a weapon set from Inspector resource).

### Combat dispatch (CMBT-03, CMBT-04)

No new scene nodes or UI components. This is pure GDScript logic in `scripts/player.gd`.

**Visible effect:** None beyond damage dealt to enemies. The indicator (CMBT-05) is the
only visual feedback for equipped weapon state.

## Copywriting Contract

This is a game UI, not a web app. Copywriting is limited to in-world labels and state text.

| Element | Copy |
|---------|------|
| Weapon slot label | `W` |
| Tool slot label | `T` |
| Empty slot state | (no text — slot is visually dimmed via modulate; no label shown when empty) |
| WeaponIndicator empty | (invisible — entire node hidden; no "no weapon" label) |
| WeaponIndicator with item (name fallback) | Item's `.name` field value — set by data, not hardcoded |
| HUD slot with item (name fallback) | Item's `.name` field value — set by data, not hardcoded |
| Error / rejection message | Existing `insert_rejected` label in `world.gd` handles bag-full feedback — unchanged by this phase |
| Destructive confirmation | None — no destructive actions in Phase 7 HUD (drop/unequip are Phase 8) |

**CTA for this phase:** None — Phase 7 has no new interactive CTA elements. Combat is triggered by
existing `hit` input action (X / Left Mouse Button). HUD slots are display-only (no click
interaction wired until Phase 8).

## Interaction States

| Component | State | Visual |
|-----------|-------|--------|
| HUD slot — empty | Default | Panel background `Color(0.12, 0.12, 0.12, 0.85)` at `modulate.a = 0.4`; no border; no icon |
| HUD slot — occupied | Equipped | Panel at `modulate.a = 1.0`; 2px gold border `Color(1.0, 0.85, 0.0)`; icon or name label |
| WeaponIndicator — no weapon | Hidden | Entire `Node2D` `visible = false` |
| WeaponIndicator — weapon equipped | Visible | `Node2D` `visible = true`; TextureRect or Label shown per texture-or-label rule |
| HUD slot — hover | Unchanged | No hover interaction in Phase 7 (Phase 8 scope) |
| HUD slot — click | Unchanged | No click interaction in Phase 7 (Phase 8 scope) |

## Anti-Patterns (Executor Must Avoid)

Taken directly from RESEARCH.md — restated here as design contract constraints.

1. **Do not parent HUD strip under InventoryUI.** InventoryUI toggles `visible` on inventory input. HUD strip must be a sibling in CanvasLayer.
2. **Do not use a shader for slot dimming.** Use `modulate.a = 0.4` — one-liner, zero compile overhead.
3. **Do not hardcode damage in player.gd.** `attack.damage = int(equipment_data.weapon.damage)` — always read from `WeaponItem.damage`.
4. **Do not subscribe to `equipment_changed` for damage caching.** Read `equipment_data.weapon` at `hit()` call time only.
5. **Do not position WeaponIndicator at `Vector2(0, 0)`.** Start at `Vector2(0, -24)`; adjust in Inspector.
