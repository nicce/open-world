## Spacing Scale

Godot uses pixel offsets and `custom_minimum_size` rather than CSS tokens.
Declared values below are the pixel constants this phase must honour, derived
from existing scenes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Corner radius on slot StyleBoxFlat (`corner_radius_*` = 4) |
| sm | 8px | HBoxContainer separation between WeaponSlotGroup and ToolSlotGroup (`theme_override_constants/separation = 8`) |
| md | 16px | HUD strip bottom margin from viewport edge (`offset_top = -80`, `offset_bottom = -16` → 16px gap) |
| lg | 24px | Collectable spawn distance minimum from player (16–32px range; 24px midpoint) |
| slot | 48px | HUD slot and IconRect `custom_minimum_size = Vector2(48, 48)` — the canonical equipment slot size |
| bag-slot | 87x80px | Bag slot `custom_minimum_size = Vector2(87, 80)` — do not change |

Exceptions:
- Popup min-size estimation: use `Vector2(120, 60)` as the fixed floor for viewport clamping (see Interaction Contract below). Source: CONTEXT.md discretion area.
- Touch-target minimum is not applicable — desktop game, mouse input only.

Source: hud_strip.tscn, inventory_slot_ui.tscn (codebase).

## Typography

All font sizes are Godot `theme_override_font_sizes/font_size` integers (pixels).
No external typeface. Weights are controlled by Godot's default font rendering;
only size variation is used — two effective "weights" are achieved through size
contrast (large = prominent, small = secondary).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| HUD slot label (W / T marker) | 11px | regular (default) | N/A — single char |
| HUD item name fallback (no texture) | 9px | regular (default) | N/A — single line |
| Inventory weight label | default (inherits) | regular (default) | N/A — single line |
| PopupMenu items | default (PopupMenu built-in) | regular (default) | N/A — managed by Godot |

Constraints for Phase 8:
- The new HUD PopupMenu uses Godot's default PopupMenu rendering — no custom
  font size override needed. Match the existing InventoryUI PopupMenu (no
  explicit font_size set in inventory_ui.tscn).
- Do NOT introduce new font sizes beyond 9px and 11px already in hud_strip.tscn.

Source: hud_strip.tscn `theme_override_font_sizes` (codebase).

## Color

All colors are Godot `Color(r, g, b, a)` values used in StyleBoxFlat and
modulate properties. No hex palette — values below are the canonical Godot
Color() representations extracted from scene files.

| Role | Godot Value | Usage |
|------|-------------|-------|
| Dominant (60%) — slot background | `Color(0.12, 0.12, 0.12, 0.85)` | HUD slot panel background (StyleBoxFlat bg_color in hud_strip.tscn) |
| Secondary (30%) — empty state | `modulate.a = 0.4` applied to parent Color(1,1,1,1) | Empty slot dimming — both bag slots and HUD slots when unoccupied |
| Accent (10%) — occupied indicator | `Color(1.0, 0.85, 0.0)` | Border colour on occupied slots only (2px border via StyleBoxFlat border_color) |
| Destructive | not applicable in Phase 8 | Drop is non-destructive in Godot context (item spawns as collectable) |

Accent reserved for:
- Occupied HUD slot border (weapon slot when weapon equipped, tool slot when tool equipped).
- Selected bag slot border in InventoryUI.
- The new HUD popup "Drop" action does NOT use a destructive accent — no red
  highlight. Drop spawns the collectable; it is a safe, reversible action. The
  gold accent is ONLY for the slot occupied-state border.

Source: hud_strip.gd `_set_slot_border()`, inventory_slot_ui.gd `_selected_style`
(codebase).

## Interaction Contract

This section defines the precise interaction behaviour that Phase 8 must implement.
It supplements the color/spacing contract with the specific states and flows.

### Context Menu — HUD Slot (new in Phase 8)

| Trigger | Condition | Result |
|---------|-----------|--------|
| Right-click WeaponSlot | weapon equipped | PopupMenu appears with "Unequip" (MENU_UNEQUIP = 0) and "Drop" (MENU_DROP = 1) |
| Right-click ToolSlot | tool equipped | PopupMenu appears with "Unequip" (MENU_UNEQUIP = 0) and "Drop" (MENU_DROP = 1) |
| Right-click WeaponSlot | slot empty | No menu shown — no action |
| Right-click ToolSlot | slot empty | No menu shown — no action |
| Inventory panel closed | any | HUD right-click still works (HUD is always visible) |

PopupMenu node: child of HudStrip root — NOT a child of InventoryUI.
Named const IDs: `MENU_UNEQUIP = 0`, `MENU_DROP = 1` (source: CONTEXT.md decisions).

### Viewport-Safe Popup Positioning

Applies to BOTH inventory_ui.gd AND hud_strip.gd popup call sites.

Algorithm (GDScript):
```
var pos: Vector2 = get_viewport().get_mouse_position()
var vp: Vector2 = get_viewport().get_visible_rect().size
var popup_min := _popup_menu.get_contents_minimum_size()
# Floor at 120x60 if get_contents_minimum_size() returns zero before first show
if popup_min == Vector2.ZERO:
    popup_min = Vector2(120.0, 60.0)
pos.x = clampf(pos.x, 0.0, vp.x - popup_min.x)
pos.y = clampf(pos.y, 0.0, vp.y - popup_min.y)
_popup_menu.popup(Rect2i(pos, Vector2i.ZERO))
```

Whether to extract this into a shared static utility or keep it inline in both
call sites is at Claude's discretion (source: CONTEXT.md discretion area).

### Unequip from HUD Slot

"Unequip" action: calls `_equipment_data.unequip_weapon()` or
`_equipment_data.unequip_tool()` and attempts `_inventory.insert(item, 1)`.
If bag is full (remaining > 0), unequip is rejected — weapon stays equipped.
Same safe-no-op contract as InventoryUI (source: CONTEXT.md decisions).

### Drop from HUD Slot

"Drop" action: unequips item, then spawns collectable at player position — bypasses
the bag entirely. Works even when bag is full (drop != unequip-to-bag).

Pattern (from CONTEXT.md + _drop_selected() in inventory_ui.gd):
1. Call `_equipment_data.unequip_weapon()` / `unequip_tool()` — gets the item back.
2. Instantiate COLLECTABLE_SCENE, set `node.item`.
3. Position: `_player.global_position + Vector2(cos(angle), sin(angle)) * dist`
   where dist is `randf_range(16.0, 32.0)` and angle is `randf() * TAU`.
4. `get_tree().current_scene.add_child(node)`.

### Popup Lifecycle

| Event | Action |
|-------|--------|
| Inventory panel closed (toggle off) | `_popup_menu.hide()` — already handled in InventoryUI `_input()`. HudStrip popup is separate and does NOT auto-hide when inventory closes (HUD is always visible). |
| HUD popup dismissed by clicking outside | Godot PopupMenu handles this automatically (hide_on_item_selection and hide_on_checkable_item_selection default true) |
| Right-click on empty HUD slot | Guard: `if item == null: return` before `_popup_menu.clear()` |

| Element | Copy |
