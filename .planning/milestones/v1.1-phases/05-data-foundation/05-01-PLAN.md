<feature>
  <name>EquipmentData Resource</name>
  <files>scripts/resources/equipment_data.gd, tests/unit/test_equipment_data.gd</files>
  <behavior>
    - equip_weapon(sword) when slot is empty: weapon field == sword, returns null, equipment_changed emitted once
    - equip_weapon(axe) when sword already equipped: weapon field == axe, returns sword, equipment_changed emitted once
    - unequip_weapon() when sword equipped: weapon field == null, returns sword, equipment_changed emitted once
    - unequip_weapon() when slot is empty: weapon field == null, returns null, equipment_changed emitted once
    - equip_tool(potion) when slot is empty: tool field == potion, returns null, equipment_changed emitted once
    - equip_tool(herb) when potion already equipped: tool field == herb, returns potion, equipment_changed emitted once
    - unequip_tool() when potion equipped: tool field == null, returns potion, equipment_changed emitted once
    - unequip_tool() when slot is empty: tool field == null, returns null, equipment_changed emitted once
  </behavior>
  <implementation>
    Create scripts/resources/equipment_data.gd:
    - class_name EquipmentData extends Resource
    - signal equipment_changed
    - @export var weapon: WeaponItem (defaults to null — typed @export Resource fields are null by default in Godot 4)
    - @export var tool: Item (defaults to null)
    - equip_weapon(item: WeaponItem) -> WeaponItem: store previous = weapon, set weapon = item, emit equipment_changed, return previous
    - unequip_weapon() -> WeaponItem: store previous = weapon, set weapon = null, emit equipment_changed, return previous
    - equip_tool(item: Item) -> Item: same pattern
    - unequip_tool() -> Item: same pattern

    CRITICAL — always mutate the field FIRST, then emit. Do NOT emit before setting the field (listeners would read stale value).

    Create tests/unit/test_equipment_data.gd:
    - extends GutTest
    - Helper: _make_weapon() -> WeaponItem and _make_item() -> Item (use .new() on the class)
    - Use watch_signals(ed) before any call, then assert_signal_emitted(ed, "equipment_changed")
    - Do NOT require scene tree — EquipmentData is a pure Resource, instantiate with .new()

    Follow TDD: write all failing tests first (RED), then implement the Resource to make them pass (GREEN).
    Run make lint && make format-check && make test after GREEN.
  </implementation>
</feature>
