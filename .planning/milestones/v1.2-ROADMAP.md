### 🕒 Phase 14: Technical Debt and Architectural Polish (Improvement Notes)
**Goal**: Address technical debt identified during v1.2 development and audit.

**Refactoring & Architecture:**
- [ ] **Signal-based Health/Hitbox**: Transition `HitboxComponent` and `HealthComponent` from direct `@export` references to a fully signal-based architecture to reduce tight coupling.
- [ ] **Spawner Component**: Consolidate enemy spawning logic from `abandoned_village.gd` and `lake_world.gd` into a reusable `SpawnerComponent` with `SpawnConfig` resources.
- [ ] **Item Identity Safety**: Ensure all item matching logic (especially in `inventory.gd`) uses unique IDs or resource pointers rather than name strings.

**UI/UX Improvements:**
- [ ] **Global Focus Management**: Extend the keyboard navigation from Phase 13 to `InventoryUI` and all other menu systems for full gamepad/keyboard playability.
- [ ] **Item Collection Feedback**: Implement missing signal handlers in `InventoryUI` to provide visual feedback (e.g., highlighting or toast) when items are collected.
- [ ] **Hover Tooltips**: Add informational tooltips to inventory and equipment slots showing item stats and descriptions.

**Performance & Safety:**
- [ ] **Process Optimization**: Replace redundant visibility/animation updates in `campfire.gd` and other interactive objects with signal-driven updates (remove polling from `_process`).
- [ ] **Safe Collections**: Refactor `collectable.gd` to use safe method checks (`has_method`) instead of hard `assert` calls to prevent crashes during player refactors.
- [ ] **Medicpack Standardization**: Convert `Medicpack` from a direct-healing object to a standard `Collectable` that adds a `HealthItem` resource to the inventory.

**Testing Expansion:**
- [ ] **State Machine Fuzzing**: Add unit tests for complex state transitions (e.g., taking damage during an attack or state transition).
- [ ] **UI Integration Tests**: Add GUT tests for inventory slot selection, dragging (if applicable), and context menu logic.
