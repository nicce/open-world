# CLAUDE.md — open-world

## Project Overview

A top-down 2D adventure/RPG game built with **Godot Engine 4.2+** and **GDScript**. Core mechanics include player movement and combat, enemy AI, inventory management, interactive objects (campfire), and area-based background music.

## Running the Game

1. Open Godot Engine 4.2 or later.
2. Import/open the project at the repository root.
3. Main scene: `res://scenes/world.tscn`
4. Press **F5** (or the Play button) to run.

There is no CLI build step. The `.godot/` cache directory is git-ignored and regenerated by the editor automatically.

## Project Structure

```
scenes/       # .tscn scene files (world, player, enemies, UI, items)
scripts/      # GDScript source (.gd files)
  resources/  # Custom Resource classes (inventory, item, weapon, health item)
components/   # Reusable sub-scenes (health_component, hitbox_component, etc.)
art/          # Sprites, tilesets, animation sheets
  characters/ # Player and enemy sprites (with normal maps)
  items/      # Weapon and key sprites
  tilesets/   # Grass, field, water, nature, house, village, floor tilesets
  animations/ # Animation frame sheets
music/        # Audio tracks (home.wav, dark_woodlands.wav)
animations/   # Animation resource (.tres) files
```

## Architecture Patterns

### Component-based composition
Reusable functionality lives in standalone sub-scenes under `components/`:
- `health_component.tscn` / `health_component.gd` — HP, damage, healing, death signal
- `hitbox_component.tscn` / `hitbox_component.gd` — area-based damage with cooldown
- `player_detection_component.tscn` — enemy awareness area
- `animation_damage.tscn` — flash effect on hit

Attach these as children of any scene that needs the behaviour.

### Signals
Systems communicate via signals rather than direct node references. Example: `health_component` emits `died` and `health_changed`; listeners respond without tight coupling.

### State machines
Player uses an explicit enum state machine (`MOVE`, `HIT`, `DEAD`) in `scripts/player.gd`.

### Autoload singleton
`BackgroundMusic` (`scenes/background_music.tscn` / `scripts/background_music.gd`) is registered as an autoload. Access it globally via `BackgroundMusic` to switch tracks by area.

## Code Conventions (GDScript)

- **Class names:** `PascalCase` via `class_name` declaration at top of file.
- **Variables / methods:** `snake_case`.
- **Exported variables:** `@export` annotation; configure values in the Inspector.
- **Node references:** `@onready var foo = $NodePath` pattern.
- **Resource types:** extend `Resource` and live under `scripts/resources/`.
- Prefer signals over polling. Avoid `get_node()` in `_process()`; cache with `@onready`.

## Testing

There is no automated test suite. Test changes by running the game in the Godot editor (F5) and exercising the affected systems manually.
